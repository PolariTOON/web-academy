Présentation de la plateforme web

Vocation de la plateforme web

Le web a été imaginé en 1989 comme un système décentralisé de documents reliés entre eux reposant sur l'internet. Une de ses vocations est de pouvoir mettre à disposition du contenu et de pouvoir le consulter depuis n'importe où, quel que soit l'appareil.

La plateforme web est l'ensemble des technologies (formats, schémas, langages, protocoles, APIs…) disponibles dans les navigateurs et à partir desquelles ces documents web sont construits. Elles sont majoritairement décrites par des standards ouverts maintenus par divers groupes de travail, indépendants ou non, mais parfois étendues par des fonctionnalités propriétaires.

Les concepts de web sémantique et de web 3.0 ne sont communément pas englobés par cette définition.

Bonnes pratiques

Les auteurs de contenus ont aussi une responsabilité dans la garantie de la bonne santé du web. Concevoir les documents dans le respect des standards, et les tester régulièrement dans les différents navigateurs et leurs différentes versions favorise l'accessibilité du web, sans entraver l'évolution de la plateforme web.

Moteurs de rendu

Les différents navigateurs majeurs actuels reposent tous sur trois implémentations distinctes appelées moteurs de rendus, même si chacun y ajoute ensuite ses propres fonctionnalités.

WebKit avec JavaScriptCore

C'est le seul moteur de rendu disponible sous iOS. Il est donc à la base de toutes les variantes de tous les navigateurs sur ce système, mais aussi de Safari sous macOS. On le retrouve aussi dans Gnome Web sous certaines distributions Linux.

Blink avec V8

C'est le moteur de rendu le plus répandu actuellement qui fait notamment tourner les versions actuelles de Chrome et Edge. Initialement dérivé de WebKit, il en garde encore certaines fonctionnalités propriétaires.

Gecko avec SpiderMonkey

C'est le moteur de rendu de Firefox, mais aussi de Tor et de Wolvic, des navigateurs plus exotiques.

Le nombre de moteurs de rendu maintenus a tendance à diminuer avec le temps. Ainsi Presto, à la base des anciennes versions d'Opera, et Trident, à la base d'Internet Explorer et des anciennes versions d'Edge ont disparu. Cependant d'autres sont en train d'émerger, comme Servo et Flow.

Technologies

Le web est une des applications majeures de l'internet, à l'instar des courriels. À sa base, l'internet repose sur des protocoles et sytèmes d'adressage comme TLS, UDP, TCP, DNS et IP. Le web y ajoute sa couche de technologies plus spécialisées, en commençant par le protocole HTTP et les adresses URL. Viennent ensuite les documents HTML, enrichis dans lesquels sont inclus ou référencés d'autres documents (HTML, PDF), certains types spécifiques de médias (images, audio, vidéos), autour desquels gravitent notamment CSS pour la mise en forme ou encore JS et WASM pour la programmation, un certain nombre d'APIs généralement exposées via JS et d'autres formats de données comme génériques comme XML ou JSON.

Présentation de HTTP

HTTP

Hypertext Transfer Protocol

Méthodes de requêtes : DELETE, GET, POST, PUT…

Codes de réponses : 200 OK, 302 Found, 403 Forbidden, 404 Not Found, 500 Internal Server Error…

HTTPS

Hypertext Transfer Protocol Secure

Échange HTTP chiffré et signé avec SSL / TLS

Garantit la confidentialité et l'intégrité des données transférées

Présentation des langages du web

HTML

Hypertext Markup Language

Langage de structuration de document

Balises (<html>, <body>, <p>…)

Aujourd'hui : HTML 5

CSS

Cascading Style Sheets

Langage de présentation de document

Permet de styliser une page

Syntaxe : div#content {color: blue;}

Aujourd'hui : CSS 3

JS

Javascript

Nom officiel : ECMAScript

Langage de programmation côté client

Permet d'animer une page

Attention : n'est pas Java

Aujourd'hui : ECMAScript 2018

XML

eXtensible Markup Language

Méta-langage d'échange de données

Permet la communication entre applications

Différents langages : XSLT, XPath, XHTML

JSON

Javascript Object Notation

Langage d'échange de données

Syntaxe : objets de JavaScript

Utilisé par certaines bases de données NoSQL
