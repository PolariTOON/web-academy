TP 1 - HTML

Ce TP consiste à réaliser la structure d'un CV en suivant le modèle dit chronologique.

Libre à vous de faire le vôtre ou bien celui de votre personnage de fiction préféré !

Et surtout : n'hésitez pas à poser des questions !

Squelette

Pour commencer, partons d'un document HTML vide :

  1. Construisez le squelette minimal de votre document HTML tel que celui-ci soit valide.

  2. Dans quelle langue est rédigé votre document ?

    Renseignez cette information au bon endroit dans le code HTML.

  3. Déterminez le titre que vous donneriez au document.

    Pour cela, posez-vous les questions suivantes : quel est le sujet du document ? quel titre aimeriez-vous voir s'afficher dans le référencement d'un moteur de recherche ? comment comprendre de quoi traite le document juste en voyant son titre, sans l'ouvrir ?

Hiérarchie

À partir du squelette construit précédemment, délimitons les différentes parties du document :

  1. Identifiez les titres des parties et sous-parties du CV et ajoutez-les au document en utilisant le niveau de titre approprié.

  2. Combien de balises <h1></h1> avez-vous utilisées ?

    Dans ce type de document, si vous n'utilisez pas de balises de sections, il ne devrait y avoir qu'une paire de balises <h1></h1>.

  3. Le lecteur du CV cherchera sans doute à contacter son auteur (on l'espère en tout cas !) :

    1. Quelle paire de balises sémantiques utiliseriez-vous pour renseigner les informations permettant de contacter le candidat : adresse postale, téléphone, adresse électronique, site web, galaxie, dimension… ? (à chercher sur l'internet)

    2. D'un point de vue hiérarchique, dans quelle partie du document placeriez-vous cet ensemble d'informations ?

    Ajoutez ces informations au document.

Contenu

Commençons à remplir les différentes parties de ce document :

  1. Dans chacune des parties du CV devra figurer une chronologie de différents événements :

    1. Quelles balises pensez-vous qu'il soit approprié d'utiliser pour réaliser une chronologie associant à chaque date ou période un ou plusieurs événements ? (à chercher éventuellement sur l'internet)

    2. Voyez-vous un problème à utiliser des balises <div></div> ?

      N'hésitez pas à demander si vous ne trouvez pas… (il y a potentiellement plusieurs réponses possibles)

    Ajoutez les différentes chronologies au document.

  2. Il est temps d'ajouter une photographie au CV :

    1. Quelle est la balise incontournable pour ce genre de donnée ?

    2. Où placeriez-vous une telle balise dans le document ?

    3. La photographie s'affiche-t-elle tel que vous l'auriez souhaité ? (chargement, placement, taille…)

    4. Si vous avez le temps, tentez de corriger vos différents problèmes en utilisant uniquement des balises et attributs HTML appropriés.

      Cela suffit-il pour tous les résoudre ?

    Laissez la photographie de côté pour l'instant : lorsque vous aurez des connaissances en CSS, il vous sera possible de corriger ces problèmes sans avoir recours à du code HTML supplémentaire (ce qui est d'ailleurs à éviter).

Validation

L'essentiel du CV a été réalisé désormais.

Vérifiez la validité de votre code HTML à l'adresse https://html5.validator.nu/ puis corrigez le maximum d'erreurs et d'alertes recensées par l'outil.
